# Larva Seconds Lost external module

This project is the Epic 01 baseline for the Scelight larva feature. It proves that an external module can be built, packaged, installed, enabled, and observed before any replay-analysis logic is added.

## SDK-style layout

- `src/` - Java 7 source code and packaged resources.
- `bin/` - compiled classes generated by the Ant build.
- `lib/` - optional third-party jars to bundle with the module.
- `release/` - generated Scelight release tree, deployment zip, manifest templates, and build properties.

The build uses the shared Scelight external module API jar from `../ScelightExtModSDK/Scelight-ext-mod-api/1.5.2/lib/` so the module stays aligned with the SDK already present in the workspace.

## Build

1. Keep the shared SDK folder in the workspace root next to this project.
2. From this project folder, run `ant BUILD_RELEASE`.
3. The build compiles with `source=1.7` and `target=1.7`, then creates:
   - `release/Scelight/mod-x/larva-seconds-lost/<version>/`
   - `release/deployment/LarvaSecondsLost-<version>.zip`
   - `release/deployment/module.xml`

## Install into a local Scelight instance

1. Edit `release/release.properties` and set `scelightFolder` to your local Scelight installation directory.
2. Run `ant INSTALL_DEPLOYMENT`.
3. The install target removes older copies of `mod-x/larva-seconds-lost` and unpacks the fresh deployment.
4. In Scelight, enable the module from the Installed Modules page.

## Observable runtime proof

When the module is enabled:

- Scelight logs module startup and shutdown.
- A new page named **Larva** appears in the main page tree.
- The page shows a short baseline message and the next replay-integration question.

## Troubleshooting

- If compilation fails, verify the API jar exists at `../ScelightExtModSDK/Scelight-ext-mod-api/1.5.2/lib/scelight-ext-mod-api-1.5.2.jar`.
- If install fails, confirm `scelightFolder` points to a directory literally named `Scelight` and that it contains one of `Scelight.exe`, `Scelight-linux.sh`, or `Scelight-os-x.command`.
- If the module is installed but not visible, check that it is enabled in Scelight's module management UI and review the application log for the module lifecycle messages.
